บทนำ

ในการเชื่อมต่อ API ของร้านนั้น ทางเราได้เตรียม RESTful API ไว้สำหรับทำการสื่อสารกับเซิร์ฟเวอร์โดยใช้ HTTP Request โดยสามารถส่ง Request พร้อมรูปแบบ Payload มาที่


RESTful Endpoint

การยืนยันตัวตน

ในการยืนยันตัวตน ท่านจะต้องนำ API Key ที่ได้จากทางระบบ Generate Key API ของท่าน โดยท่านจะต้อง encode Key API ของท่านด้วย Base64 อีกครั้งก่อนทำการเชื่อมต่อ และตั้งค่า Header ของ Payload ในทุก Request ของท่านดังนี้


Authorization: Basic base64(YOUR_API_KEY_HERE)

ตรวจสอบข้อมูลผู้ใช้

ในการตรวจสอบข้อมูลผู้ใช้ของท่าน ท่านสามารถทำการส่ง Payload ดังกล่าวมายังเซิร์ฟเวอร์ได้ดังตัวอย่างต่อไปนี้


GET https://api.peamsub24hr.com/v2/user/inquiry

โดยหาก Payload ของท่านนั้นถูกต้อง เซิฟเวอร์จะทำการตอบท่านดังตัวอย่างต่อไปนี้


{
	"statusCode": 200,
	"data": {
		"username": "mydear0909", // ชื่อผู้ใช้ของท่าน
		"balance": "0.00", // ยอดเงินคงเหลือ
		"rank": 1 // ยศ​ โดย 1 = ตัวแทนจำหน่าย; 3 = ตัวแทน VIP
	}
}

ตัวอย่าง cURL Request:


curl --request GET \ --url https://api.peamsub24hr.com/v2/user/inquiry \
  --header 'Authorization: Basic base64(YOUR_API_KEY_HERE)' \
  --header 'Content-Type: application/json' \

ซื้อสินค้าแอพพรีเมียม

ในการซื้อสินค้าแอพพรีเมียมที่ทางร้านได้ทำการเปิดขายนั้น ท่านจะต้องทำการส่ง Payload ดังกล่าวมายังเซิร์ฟเวอร์ได้ดังตัวอย่างต่อไปนี้


POST https://api.peamsub24hr.com/v2/app-premium

JSON Body:
{
"id": 1, // ไอดีของสินค้าที่คุณต้องการซื้อ
"reference" : "ABCD" // reference ไว้เพื่อเรียกใช้ในอนาคต โดยคุณสามารถใช้ order id ของร้านคุณหรืออื่นๆ เพื่ออ้างอิงประวัติการซื้อได้อย่างถูกต้อง
}

โดยหาก Payload ของท่านนั้นถูกต้องและการซื้อสินค้านั้นเสร็จสมบูรณ์ เซิฟเวอร์จะทำการตอบท่านดังตัวอย่างต่อไปนี้


{
	"statusCode": 200,
	"data": "TEST PRODUCT" // ข้อมูลของสินค้า โดยท่านสามารถบันทึกข้อมูลสินค้านี้ได้เลย
}

และหากเกิดปัญหาในการซื้อสินค้า เซิฟเวอร์จะทำการตอบท่านดังตัวอย่างต่อไปนี้


{
	"statusCode": 400,
    "error": "Bad Request",
	"message": "ยอดเงินของคุณไม่เพียงพอ"
}

ตัวอย่าง cURL Request:


curl --request POST \
  --url https://api.peamsub24hr.com/v2/app-premium \
  --header 'Authorization: Basic base64(YOUR_API_KEY_HERE)' \
  --header 'Content-Type: application/json' \
  --data '{
	"id": 1,
	"reference": "a"
}

POST https://api.peamsub24hr.com/v2/app-premium/history

// โดยหาก array references เป็น array เปล่า เซิร์ฟเวอร์จะตอบกลับทุกประวัติของท่านที่เคยซื้อผ่านทาง API
}

โดยหาก Payload ของท่านนั้นถูกต้องและสามารถดึงประวัติการซื้อสินค้าได้ เซิฟเวอร์จะทำการตอบท่านดังตัวอย่างต่อไปนี้

เปิดเคลมสินค้าแอพพรีเมียม

ในการเรียกใช้ประวัติการซื้อสินค้าแอพพรีเมียม ท่านจะต้องทำการส่ง Payload ดังกล่าวมายังเซิร์ฟเวอร์ได้ดังตัวอย่างต่อไปนี้


POST https://api.peamsub24hr.com/v2/app-premium/claim
JSON Body:
{
  "reference": "ABCD123", // นำไอดี reference ที่ท่านเคยสร้างและเก็บไว้ตอนซื้อสินค้าเพื่อเปิดเคลม
  "status": "wrong_password", // สถานะสินค้า
  "description": "ทดสอบการเคลมสินค้า", // อธิบายปัญหาที่พบ โดยจะใส่หรือไม่ใส่มาก็ได้
  "callbackUrl": "YOUR_CLAIM_CALLBACK_API_HERE" // callback url ของเว็บไซต์ท่าน
}

สำหรับสถานะในการแจ้งเคลม ท่านสามารถแจ้งเคลมตามสถานะ โดยใช้ค่า enum ดังต่อไปนี้


{
  wrong_password // แจ้งเคลมรหัสผ่านผิด
  incorrect_pin // PIN ไม่ถูกต้อง
  youtube_premium_disconnect // Youtube Premium หลุด
  netflix_screen_disconnect // โดนมั่วจอ Netflix หรือแอพอื่นๆ
  others // พบปัญหาอื่นๆ ติดต่อแอดมินหรือพิมพ์แจ้งใน description มาได้เลย
}

ในส่วนของ callback url นั้น ท่านจะใส่หรือไม่ใส่ก็ได้ แต่เราแนะนำให้ท่านใส่ callback url เพื่อติดตามผลการแก้ไขสินค้าอย่างมีประสิทธิภาพ
โดยหากท่านไม่ได้ใส่ callback url ไว้ ท่านจะต้องคอยดึงประวัติสินค้าจากเว็บเราตลอด ณ​ ตอนที่ลูกค้าเรียกดูสินค้า เพื่อให้ข้อมูลรายละเอียดสินค้านั้นล่าสุดและอัพเดตอยู่ตลอดเวลา (ซึ่งทางเราไม่แนะนำ เพราะอาจจะส่งผลต่อประสิทธิภาพของเว็บไซต์ท่านได้)


โดยหาก Payload ของท่านนั้นถูกต้องและการเคลมนั้นเสร็จสมบูรณ์นั้น เซิฟเวอร์จะทำการตอบท่านดังตัวอย่างต่อไปนี้


{
  "statusCode": 200,
  "data": {
    ticketId: 'ABCDEFG' // ไอดีเปิดเคลม โดยท่านจะต้องบันทึกไอดีนี้ไว้ถ้าท่านแนบ callback url มาด้วยเพื่อใช้รับ callback อีกครั้ง
  }
}

และหากมีเราได้การแก้ไขสินค้าของท่านเรียบร้อยแล้ว ทางเราจะทำการส่ง POST Request กลับไปหา callback url ของท่านดังตัวอย่างต่อไปนี้


POST YOUR_CLAIM_CALLBACK_API_HERE
JSON Body:
{
  "ticketId": "ABCD123", // ไอดีการเปิดเคลม
  "status": "success", // สถานะการเปิดเคลม
  "prize": "สินค้าที่เคลมแล้ว" // รายละเอียดสินค้า 
  // โดยท่านจะต้องอัพเดตรายละเอียดสินค้าที่ให้ไปใหม่นี้ในฝั่งลูกค้าของท่าน
}

โดยหากท่านได้รับการ callback อย่างถูกตัองแล้ว ท่านจะต้องตอบเซิร์ฟเวอร์ของเราดังตัวอย่างต่อไปนี้เพื่อยืนยันการรับ callback


HTTP Status: 200
{
  "statusCode": 200,
}

ตัวอย่าง cURL Request:


curl --request POST \
  --url http://api.peamsub24hr.com/v2/app-premium/claim \
  --header 'Content-Type: application/json' \
  --header 'User-Agent: insomnia/11.2.0' \
  --data '{
  "reference": "ABCD123", 
  "status": "wrong_password",
  "description": "ทดสอบการเคลมสินค้า", 
  "callbackUrl": "YOUR_CLAIM_CALLBACK_API_HERE"
}'
